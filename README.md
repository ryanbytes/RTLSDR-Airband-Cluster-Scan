## Overview

RTLSDR-Airband receives analog radio voice channels and produces
audio streams which can be routed to various outputs, such as online
streaming services like LiveATC.net. Originally the only SDR type
supported by the program was Realtek DVB-T dongle (hence the project's
name). However, thanks to SoapySDR vendor-neutral SDR library, other
radios are now supported as well.

## Documentation

User's manual is now on the [wiki](https://github.com/rtl-airband/RTLSDR-Airband/wiki).

## Clustered Scan Mode

`mode = "cluster_scan"` groups configured channel frequencies into RF clusters
that fit inside the usable SDR passband. Each cluster tunes the SDR once and
demodulates all channels in that cluster with the normal multichannel path, then
hops to the next cluster when no active squelch remains after
`cluster_hang_time_ms`.

Cluster sizing uses `sample_rate - 2 * edge_guard_hz`; the default
`edge_guard_hz` is 150000 Hz. Optional `max_cluster_width_hz` can further limit
cluster width. Use `rtl_airband -P -c <config>` or
`cluster_scan_print_plan = true` to print the generated plan without starting
SDR hardware. See `config/cluster_scan.conf` for an example.

Clustered scan reduces retunes and improves the probability of catching short
calls within the active RF window. It still misses traffic outside the currently
tuned cluster. Too-wide clusters or channels near passband edges may reduce
sensitivity.

## Web GUI

The optional web GUI is a cross-platform static browser dashboard. Configure
`rtl_airband` to write `status.json`:

```conf
gui_status_filepath = "/path/to/gui/web/status.json";
gui_status_interval_ms = 1000;
```

Then serve `gui/web` with any static file server and open it in a browser. See
`gui/README.md` for a minimal `python3 -m http.server` example.

## Scan TUI Colors

The foreground scan display (`rtl_airband -f`) supports configurable color
themes. The default `scan_tui_theme = "bright"` uses brighter idle rows than the
original dim gray display. Other presets are `high_contrast`, `monochrome`,
`deuteranopia`, `protanopia`, and `tritanopia`.

Individual colors can be overridden with `scan_tui_colors`. Values are ANSI SGR
number strings without the `ESC[` prefix or trailing `m`, for example
`idle_row = "1;37"` or `open_status = "1;30;42"`. See
`config/scan_tui_themes.conf` for an example.
